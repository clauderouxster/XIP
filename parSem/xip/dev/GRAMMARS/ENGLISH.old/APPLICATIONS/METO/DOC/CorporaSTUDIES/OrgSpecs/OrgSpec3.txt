1. Comment les reconnaitre (pour détail voir ci-dessous)

 1.1 indices forts : 

a) détermination : pronom possessif > det indéfini (mais rare) > det défini (attention, possibilité de confusion avec litéral)

gestion des déterminants :
	-  enlever advs DETD(mot, ADV:...) (erreurs la plupart du temps)
	-  possessifs : ne pas prendre en compte le 3P (it): à enlever avec le lemme, on garde tous les autres. Sinon le deter possessif a l'air tres discriminant pour les products, même s'il se rencontre tout seul, on peut dire que l'EN est une org-for-product.
	-  dets indéfinis : normalement pas de pb, c'est product.
	-  dets définis : discriminant si associés avec marque du pluriel et adjectif mais attention, il 			existe des cas ou c'est du littéral.
	Dans quels cas ne pas considérer le det comme indice de product :
		si det a une majuscule
		si det + orghead/orgend dans l'EN 
		si det + POSSES(workstation,board,store,supermarket,superstore,skill, monopoly,vehicle,
		recorder) alors literal.rq : pas bcp de POSSES pour les products (aucun ds corpus 			d'entrainement).
		si det tout seul 
		si det + EN en SUBJ d'un verbe à forte connotation literal (ca sera fait par défaut)
		si det + nom de NADJ de country avant (souvent ds un MOD)
		si det + PREP_of(part,staff, subsidiary,board,director general,symbol,president) 
		si det + PREP
	---> hormis tous ces cas, on peut dire que c'est product.

b) marque du pluriel : attention xip ne gère pas toujours le nombre pour les Noms Propres.

 1.2 autres indices :

rôle grammatical : certaines PREP > QUALIF(org,adj) ou MOD_METORECT(org,adj) > OBJ-N(vb,org-prod) > SUBJ-N(vb,org-prod) > MOD_METOREG(mot,org)


travail lexiques :

pour les verbes en obj et subj
pour les modifieurs

a) les bonnes PREP :

PREP_OF(price/door/glass,org-prod)
PREP_AT(look,org-prod)
PREP_BETWEEN(mot,org-prod) --->lexique : si mot=collision --->product, si mot=agreement,etc.--->literal
PREP_WITH(mot,org-prod) attention, confusion avec members, prendre avec : problem 
PREP_IN(mot,org-prod)

les bonnes combinaisons : (pl optionnel)

- OBJ +DET +PL +QUALIF ou MODadj

- SUBJ + DET +QUALIF ou MODadj

- si que MODREG, c'est mieux si +DET +PL (---> à retrouver pour le mot recteur, e.g, "IBM users")

- si que DETdéfini, c'est mieux si +PL

- POSSES(org,mot) : il faut que mot=artifact (WN?) et c'est mieux si DET (sur mot)
